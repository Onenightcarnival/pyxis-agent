# 020 全站口吻扫盘：bullet 密度 + 结构化 + 精炼

## 目的

2026-04-22 用户定义了想要的文档风格：**内容精炼、言简意赅、结构化程度
高、配合 bullet point 简明扼要说明**。我在本规格里把风格规则落地成可
执行的编辑动作，对全站逐页过一遍，按合适的力度修改。

## 风格规则（写作约定，加进我的工作清单）

- **默认 bullet**——列事实 / 选项 / 特性一律 bullet，不堆成长句
- **prose 限 1 句**——每段最多一句连贯句做语境；多了就拆成 bullet
- **短语代替完整句**——bullet 里 `→` / `—` / `:` 做分隔比主谓宾更密
- **表格只在 2+ 项共享属性时用**——否则 bullet 更轻
- **header 多给**——宁可一页 6 个 H2 也别堆一面墙
- **保留"一句话讲透因果"的 prose**——如 step.md 里"行为会变差：LLM
  先塞一个模糊的一句话，再用关键点'自圆其说'"，这种比 bullet 有力

## 验收

- 总行数明显下降（目标全站 -20% 以上）；信息零损失（事实、契约、代码
  示例全在）。
- `uv run --group docs mkdocs build --strict` 零警告。
- `uv run ruff check` / `ruff format --check` / `pytest` 全绿。
- 公共面（README / docs）的事实与当前 `src/` 对得上（口吻改，不改
  契约）。

## 不做

- 不动 `CLAUDE.md` / `specs/` / `apps/*/README.md`（场景不同）。
- 不改 cookbook 自动生成链路（触及 `examples/*.py` 的 docstring 是
  另一条独立迭代）。
- 不改 `src/` 源码或任何 API 签名。
- 不加插图 / GIF（需要真的截图素材，等 apps 补）。
