# 014 文档站

## 目的

框架已经 public，需要一个在线站点，让读者顺着 **定位 → 概念 → API → 迭代档案**
一条路看懂 pyxis。站点复用已有资产（CLAUDE.md、specs/、源码 docstring），
不另写一份平行文档。

## 技术选型

- MkDocs Material（instructor 同款骨架）
- mkdocstrings[python]：源码 docstring 自动渲染 API 参考
- mkdocs-gen-files + mkdocs-literate-nav：把 `specs/`、`CHANGELOG.md`、
  `ROADMAP.md` 作为虚拟文件纳入站点，不做物理复制
- 单语中文（符合规约 006），`theme.language = zh`
- 部署：GitHub Actions → GitHub Pages（仓库 Pages source 已设为 Actions）

## 站点骨架

导航用 MkDocs Material 的 `navigation.tabs`——顶部横 tab + 左侧竖栏，横纵分明。

- **首页**：agent-for-machine 定位、核心机制、"故意不做"
- **概念指南**：7 篇——哲学与定位、Step、Flow、Tool、Hook、human-in-the-loop、
  MCP。每篇可独立跑一个 snippet
- **API 参考**：按模块一页，`::: pyxis.<module>` 自动生成
- **附录**：对比其他框架、接入 Langfuse、更新日志

specs/ 和 ROADMAP 是团队内部资料，不对用户展示（2026-04-21 用户反馈：
读者不关心开发迭代档案），留在仓库里即可。

## API 草图

- `mkdocs.yml` + `docs/` 为站点源；现有 `docs/langfuse.md` / `docs/对比.md`
  就地复用，按新骨架重命名
- `docs/_hooks/gen_api.py`：构建期按模块生成 API 参考页；
  `docs/_hooks/gen_specs.py`：把 CHANGELOG 挂进站点并重写相对链接
- `pyproject.toml` 加 `[dependency-groups] docs = [...]`
- `uv run --group docs mkdocs serve` 本地预览
- `.github/workflows/docs.yml`：push main → `mkdocs build --strict` →
  `actions/deploy-pages`

## 验收

- `uv run --group docs mkdocs build --strict` 零警告
- 首页展示"agent-for-machine"定位与"故意不做"段
- API 参考能检索到 `Step` / `Tool` / `trace` / `mcp_toolset` 等核心符号
- 顶栏 tab 切换时左侧栏同步变化（概念 / API 参考 / 附录 各有自己的竖栏）
- 单元测试与集成测试不受影响

## 不做

- 不做英文版（规约 006）
- 不做版本化文档（1.0 前不折腾）
- 不做交互式 playground / 在线执行
- 不做 GA / 埋点 / 自建搜索
