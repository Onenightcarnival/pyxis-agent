# 016 文档站与仓库的映射一致性；apps 可见化

## 目的

2026-04-21 用户反馈：从文档站跳到 repo 的体验衔接**不够顺**——参照物是
instructor（docs 的 Concepts / Cookbook / Integrations 在 repo 里都能一眼
找到对应目录）。作为 PM 我把"违和感"分解成三条，并修掉前两条：

1. **apps/ 在文档站里完全缺席**。这是仓库里视觉最亮的资产（chat-demo
   与 mcp-demo 各自带前端、各自已有详尽 README），但 `mkdocs.yml` 没挂任何
   入口。访客看不到 → 下载不试 → 映射错位第一源。
2. **文档的"下一步"指引不够清**。`docs/index.md` 结尾的"从哪里开始看"
   三条项目列表并列，缺少**按意图分叉**（想试 vs 想读 vs 想查 API）。
3. **repo 根与文档站 nav 的对应关系未公示**。读者扫一眼根目录
   （specs/、ROADMAP、apps/、examples/）不知道哪些是给用户看的。

## 站点与信息架构调整

- 新增 **Demos** 一级 tab（放在 Cookbook 与 API 参考之间），三页：
  - `docs/demos/index.md`：两个 demo 的对照卡 + 决策提示（想看什么挑哪个）。
  - `docs/demos/chat-demo.md`：Chat / Inspect 双视图切换的卖点 + 截图/GIF
    占位 + 链到 `apps/chat-demo/` 与其 README。
  - `docs/demos/mcp-demo.md`：native + stdio-MCP + HTTP-MCP 三源混合
    注册的可视化卖点 + 链到 `apps/mcp-demo/` 与其 README。
- 这些页**不复写** `apps/*/README.md` 的全部细节，只承担"文档站访客
  在几秒内决定要不要点进去"的路由职责。权威说明仍在各 app 的 README。
- `docs/index.md` 结尾"从哪里开始看"改为三条意图分叉：看得见的（Demos）、
  读得进去的（概念 / 哲学）、照着抄的（Cookbook）。
- `README.md` 加一小节 **"仓库结构 ↔ 文档站"**：把 `src/` `examples/`
  `apps/` `docs/` `specs/` `ROADMAP.md` 对应到文档站的哪一级 tab 或
  说明其内部性。一个 ~8 行表格。
- `CLAUDE.md` "目录" 段同步写上 `docs/demos/`。

## 验收

- `uv run --group docs mkdocs build --strict` 零警告。
- 文档站顶栏依次出现：首页 · 概念 · Cookbook · **Demos** · API 参考。
- Demos 下三页可达；每页末尾有"源码与 README"指回 `apps/<name>/`。
- `README.md` 的"仓库结构 ↔ 文档站"段存在且为真（对应关系验证过）。
- `pytest` 全绿；`ruff check` clean（无源码改动，应天然成立）。

## 不做

- 不移动 `specs/` / `ROADMAP.md`（SDD 流程依赖其根目录可见性）。
- 不重构 `docs/concepts/` 页面本体——模块式罗列在 ≥ v1.0 前够用。
- 不新建 "Getting Started" 顶级 tab（`index.md` 已承担此职能）。
- 不拍/做真截图——用 emoji + 文字描述占位，未来再补。
- 不动 `src/`、`examples/`、`apps/` 的代码。
