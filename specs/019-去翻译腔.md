# 019 去翻译腔：原语 / 构件 → 概念

## 目的

2026-04-21 用户反馈：概念 tab 里 "原语" 读着不地道——这是 "primitive"
的直译，中文 CS 文献里虽然有（如"同步原语"），但放在 agent 框架的
科普向文档里显得翻译腔。用户更喜欢"概念"。

扫了一圈，同属一类的"**翻译腔技术词**"还有 "构件"（component）。
一次清掉。

## 受影响的位置

- `docs/concepts/index.md`: "原语" ×2、"构件" ×1
- `docs/concepts/step.md`: "构件" ×1
- `CLAUDE.md`: "原语" ×3（内部 PM 文档，保持与用户视线一致，防止
  未来迭代里再被注入回 docs）

`specs/004-async.md` 里的历史用法不动——历史档案。

## 改法

- "原语" → "概念"
- "最小的构件" → 具体情境改写（一般是"最基本的一个概念"/"最基本
  的一件事"），不硬替，避免"最小的概念"这种别扭句。

## 验收

- `docs/` 与 `CLAUDE.md` 不再出现"原语" / "构件"。
- 替换后语句仍然通顺、仍然简短（不因换词撑长）。
- `uv run --group docs mkdocs build --strict` 零警告。
- `uv run pytest` / `ruff check` 全绿。

## 不做

- 不动 `specs/004-async.md` 等历史 spec。
- 不扩大到扫其他词（如"判别式联合"——那是 Pydantic 的术语，保留精度）。
- 不重写句子结构，尽量就地换词。
