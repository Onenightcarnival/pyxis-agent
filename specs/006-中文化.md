# 006：项目语言规约 —— 中文化

## 目的

本项目从此以中文为主语言。用户（读者、维护者、未来贡献者）默认读中文。
英文保留在不适合翻译的地方：Python 标识符（遵守 PEP 8）、commit message
的工具类前缀（`feat:` / `fix:` / `chore:` / `docs:`）、配置文件的 key、
第三方库提供的字符串。除此之外所有散文必须为中文。

## 规约

| 位置                           | 语言                             |
|--------------------------------|----------------------------------|
| `README.md`、`CLAUDE.md`       | 中文                             |
| `CHANGELOG.md`、`ROADMAP.md`   | 中文                             |
| `specs/*.md`                   | 中文                             |
| 源码模块顶层 docstring         | 中文                             |
| 源码类/函数 docstring          | 中文                             |
| 源码抛出的异常消息             | 中文                             |
| 示例 `examples/*.py` 的散文    | 中文（docstring、print 输出）    |
| commit message 正文            | 中文                             |
| Python 标识符（变量/类/函数名）| 英文（PEP 8）                     |
| 测试函数名                     | 英文（`test_xxx` 便于匹配）      |
| 测试的 docstring               | 中文（可选，但存在则必须中文）   |

## 验收

- `tests/test_language_policy.py` 断言 `README.md`、`CLAUDE.md`、
  `CHANGELOG.md`、`ROADMAP.md`、每个 `specs/*.md`、每个 `src/pyxis/*.py`
  均至少含 **100** 个中日韩统一表意文字（`U+4E00–U+9FFF`），防止未来迭代
  无意中把这些文件回退成英文。
- 所有既存散文立即重写为中文；一次 commit 搞完，避免半吊子状态。
- `CLAUDE.md` 的开发流程章节把"保持所有文件中文化"作为显式步骤记入。

## 不做

- 不重写 Python 标识符或重命名 API（跨 commit 的破坏性重构与语言迁移必须
  正交，否则 review 无法进行）。
- 不翻译 `uv.lock`、`.env.example` 的 key、`pyproject.toml` 的字段名。
- 不翻译 commit 历史的旧消息——git 历史只许向前。
