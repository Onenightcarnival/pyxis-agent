# 018 demos 页口吻回调

## 目的

2026-04-21 用户反馈：规格 016 新加的 `docs/demos/` 三页"AI 味 + 自证式
描述"重，看着违和——应该像规格 015 定的"平铺直叙、口语化"那样。

重读三页的问题：

- `chat-demo.md` 有 `## 为什么这个 demo 值得看`、`Chat view 不是……的
  道歉——它是……`、`Inspect view 才是 pyxis 的主场`、`可审计 / 可断言
  / 可入库换来的东西……`——**对偶 + 自证 + 喊口号**三件套。
- `mcp-demo.md` 有 `为什么这个 demo 值得看`、`换一种说法：可审计 +
  简洁这两件事在同一个屏幕上被同时演示`——同款病。
- `index.md` 开头 `apps/ 不一样：带前端、带服务进程、能在浏览器里看
  agent 每一步怎么被填出来的`——对偶 + 动词文艺化。

写 demos 页时我在脑子里把它当成"pitch page"去打磨，不自觉就拿起了
营销话术。其实它的职责只有一条：**告诉访客这两个 app 各是什么、要
不要点进去跑一下**。

## 改写原则（沿用规格 015）

- 去掉 `值得看` / `主场` / `落成` / `换来的东西` 这种动词文艺化。
- 删掉所有"不是 X、而是 Y"对偶构造。
- 不给 demo 页发"为什么值得看"的段落标题——直接描述它是啥。
- 介绍句尽量短；口吻偏同事之间聊"这俩 app 干嘛的"那种。
- 权威细节继续指向 `apps/<name>/README.md`，demos 页不重复。

## 验收

- `docs/demos/` 三页不再有上述病句。
- 每页篇幅比现在**更短**（目标 -30% 左右）。
- `uv run --group docs mkdocs build --strict` 零警告。
- `uv run pytest` / `ruff check` / `ruff format --check` 全绿。

## 不做

- 不改信息架构（Demos tab 已在 016 落地，不动）。
- 不改任何其他页（这次问题仅出现在 demos/，其他页在 015 时已洗过）。
- 不改 `apps/*/README.md`——那边不在用户视线里。
