# 015 文档口吻、可观测性重定位、信息去冗

## 目的

2026-04-21 用户以使用者视角的反馈：

1. 主要通过 GitHub Pages 文档站了解 pyxis，看不懂才翻源码——
   目前文档与源码的理解 GAP 偏大。
2. 文案"AI 味儿"重，对偶句（"不是 X、而是 Y"）与"自证式"段落过多。
3. 生产只用无侵入接入 Langfuse，自带 `trace()` 几乎不碰。
4. 整个项目对用户可见的信息有不少冗余，希望压一压。

作为 PM 我把这四条一次性落到文档和项目结构上。

## 三条编辑原则

- **平铺直叙**：去掉"不是 X、而是 Y"对偶。每页此类构造 ≤ 2 次。
- **去自证**：不喊口号、不反复宣告"我不是 X"、不堆"好处：…"列表；
  "为什么不 X"整段尽量压成一句，决策点只留在 philosophy.md。
- **去冗**：用户能从 git log / mkdocstrings 拿到的东西，文档不再抄一遍。

## 站点与信息架构调整

- **可观测性重定位**：`docs/langfuse.md` → `docs/concepts/observability.md`，
  从附录升为概念指南里的正章；首页核心概念表不再列 `trace()`。
  `trace()` / `StepHook` 降为"测试 + 自己要接 Prometheus/OTel 才用"。
- **"故意不做"清单去重**：权威版本只放在 `docs/concepts/philosophy.md`；
  `docs/index.md` / `CLAUDE.md` / `README.md` / `ROADMAP.md` 里不再重复
  列表，改成一行指引。
- **CHANGELOG 废弃**：`CHANGELOG.md` 删除、mkdocs 导航移除 "更新日志"、
  `gen_specs.py` 删除。变更历史由 git log + GitHub Releases 承担。
- **站点导航四个顶级 tab**：
  - 首页（含首页与"对比其他框架"两页）
  - 概念
  - Cookbook（新增：`examples/*.py` 自动渲染为 recipe 页，由
    `docs/_hooks/gen_cookbook.py` 生成。文件本身就是 recipe，不写第二份
    文案——改源文件就是改 cookbook）
  - API 参考
- **README 瘦身**：去重 30 秒 demo、核心概念表等文档站已讲过的内容；
  README 定位改为"一屏讲清定位 + 指向文档站"。

## 验收

- `uv run --group docs mkdocs build --strict` 零警告。
- 概念页结尾不再有"实际对象长这样"小节——API 参考由 mkdocstrings 从
  源码 docstring 自动渲染，不需要在概念页重复一遍。
- "故意不做"完整列表在 `philosophy.md` 里只出现一次；其他地方指向它。
- 仓库根目录不再有 `CHANGELOG.md`；mkdocs 导航里没有 "更新日志"。
- Cookbook tab 每个 recipe 页能正确渲染（docstring 做 intro、源码嵌 code block、
  "源码" 链接到 GitHub）；新增 `examples/*.py` 触发 `docs` workflow 重建。
- `pytest` 全绿；`ruff check` clean。

## 不做

- 不改源码 API（`trace()` / `StepHook` / `run_traced()` 保留）。
- 不退役 `specs/` 活跃目录——它不上文档站，属于内部历史归档，不算
  用户视角的冗余。
- 不改 examples/ 与 apps/。
- 不做英文版。
