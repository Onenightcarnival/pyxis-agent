# 023-公共面收敛：砍掉配套生态

## 目的

砍掉三条跟"配套生态"相关的线——provider 工厂、trace/hooks 观察体系、
`Client` Protocol 公共化。`@step(client=...)` 改为吃原生 OpenAI SDK
或 instructor 实例；观察由 Langfuse / OpenTelemetry / APM instrument
OpenAI SDK 这一层完成。

## 新契约：`@step`

`@step(*, output, model, client, params=None, max_retries=0)`

- `client` **必填**（无默认值）。吃两种东西之一：
  - `openai.OpenAI` / `openai.AsyncOpenAI` 实例 —— `@step` 内部懒 patch
    成 instructor，结果缓存复用。
  - `instructor.from_openai(...)` 返回的已补丁实例 —— 直接使用。
- `params: dict[str, Any] | None = None`：透传给 provider API
  （`temperature` / `max_tokens` / `seed` / `top_p` / `stop` / ...），
  pyxis 内不做枚举或校验。
- sync step 拿到 async client（或反之）→ 明确 `TypeError`，不隐式转换。

## 砍掉的公共面

- `pyxis.providers` 整个模块（`openrouter_client`、`openai_client`）。
- `InstructorClient` 公共类 —— 降为 `@step` 内部的懒 patch 工具。
- `Client` / `AsyncClient` Protocol 的公共 export —— 降为内部契约，
  仅 `FakeClient` 共用。
- `Message` 类型别名（`dict[str, str]`）—— 是实现泄漏，用户从不接触。
- `Usage` / `CompletionResult` / `_extract_usage` —— 观察工具的活。
- `trace()` / `TraceRecord` / `Trace` —— `src/pyxis/trace.py` 整个删除。
- `StepHook` / `add_hook` / `remove_hook` / `clear_hooks` ——
  `src/pyxis/hooks.py` 整个删除。
- `set_default_client` / `get_default_client` —— 没有 pyxis 特有 client
  需要管理了；"全局默认" 是 anti-pattern。
- 零传 `client=` 的默认路径 —— `client` 必填，不再懒构造。

## 保留的公共面

`step` / `flow` / `Tool` / `@tool` / `Step` / `AsyncStep` / `Flow` /
`AsyncFlow` / `FakeClient` / `FakeCall` / HITL 六件套（`ask_human` /
`finish` / `run_flow` / `run_aflow` / `HumanQuestion` / `FlowResult`）/
MCP 四件套（`MCPServer` / `StdioMCP` / `HttpMCP` / `mcp_toolset`）。

## 验收

- `uv run ruff check && uv run ruff format && uv run pytest` 全绿。
- 单元测试零网络；集成测试删除所有 usage / trace 相关断言。
- `examples/` 全部改用原生 `OpenAI(...)` / `AsyncOpenAI(...)` 构造 client
  并显式传入。`evals_with_trace.py` → `evals.py`：去掉 trace 依赖，用
  普通 Python list 累加 + 手写 `json.dumps` 落盘 JSONL（PM 判断：
  FakeClient 不适合 evals——evals 要真跑 LLM，不是回放预置响应）。
  `with_langfuse.py` 化简为"换一行 import"。
- 文档：`docs/concepts/observability.md` 重写为"pyxis 不做观测，这是
  常见接入模式"；`docs/concepts/philosophy.md` 的"故意不做"段加一条
  "永远不让用户手写 messages"；CLAUDE.md 顶部可观测性段同步；README
  的 quick start 示例更新。

## 不做

- 不发明任何 "pyxis 容器"包住 `sync + async` 两个 client
  （上一轮讨论中提过的 `pyxis.Client(sync=..., async_=...)` 否决）。
- 不为非 OpenAI-compatible provider 发明适配层 —— 用户自己
  `instructor.from_anthropic(...)` 然后传进来。
- 不提供任何形式的"全局默认 client"。
- 不保留 `Message` 类型别名 —— 内部直接用 `list[dict[str, str]]`。
- 不提供 hook / middleware 协议 —— 用户在 `@step` 外再套装饰器即可。
